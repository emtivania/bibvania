1. Sobre o Projeto

O projeto BibVania é um sistema web de gerenciamento de biblioteca escolar de código aberto, projetado para funcionar 100% no navegador — sem servidor próprio, sem instalação, de forma gratuita. Oferece catálogo público do acervo, controle de empréstimos, devoluções, reservas online e cadastro de livros assistido por inteligência artificial.

O projeto nasceu na EMTI Professora Maria Vânia Farias Linhares (Mucambo, CE) e foi construído por Ruan Oliveira Lima para atender às necessidades reais de uma biblioteca escolar pública. Mais tarde, foi projetado para ser replicável: qualquer escola ou biblioteca comunitária pode gerar sua própria instância personalizada usando o BibMaker, o construtor de bibliotecas incluso no projeto, sem precisar de conhecimento técnico.

O sistema é composto por múltiplas páginas com funções distintas: um catálogo público acessível a todos os alunos, uma Área do Bibliotecário completa para o bibliotecário, um arquivador digital de documentos com assinatura criptográfica (BibFiles) e o próprio BibMaker para geração de novas instâncias em formato ZIP pronto para publicação no GitHub Pages.

Tecnologias utilizadas

  • Frontend: HTML5, CSS3 e JavaScript puro — sem frameworks, sem ferramentas de build
  • Banco de dados / Auth / Realtime: Supabase (PostgreSQL com RLS)
  • Hospedagem: GitHub Pages (gratuito)
  • Classificação por IA: Groq API — analisa a foto da capa e classifica o livro automaticamente (chave gratuita)
  • Upload de capas: ImgBB API — armazena as imagens das capas na nuvem (chave gratuita)
  • Armazenamento de PDFs: Internet Archive S3 — usado pelo BibFiles para arquivar documentos com persistência de longo prazo
  • Assistência com IA: Claude (Anthropic) — usado como assistente de desenvolvimento do projeto via upload do ZIP
  • Leitor de tela: Web Speech API nativa do navegador, integrada ao BibSpeech
  • Assinatura digital: Web Crypto API (ECDSA P-384) para autenticidade dos arquivos no BibFiles

Páginas do sistema

  • index.html — Catálogo público em carrossel: busca textual, filtros por categoria, busca por voz, reservas com modal de avisos, lightbox de capas, badges de disponibilidade, Realtime, modo escuro, leitor de tela
  • admin.html — Área do Bibliotecário completo: cadastro, empréstimos, devoluções, reservas, acervo, pessoas e relatórios (requer login)
  • login.html — Autenticação segura do bibliotecário via Supabase Auth
  • acervo.html — Lista completa do acervo em formato de lista de texto (um bloco por livro com Título, ID e linhas para Autor, ISBN, Categoria, Prateleira, Palavra-chave, Qtd, Cadastro, Capa e Download — campos vazios sem indicador), busca em tempo real, lightbox de capas, exportação em tabela (PDF landscape), barra de acessibilidade com menu de atalhos
  • bibmaker.html — Construtor com 2 abas: cadastro de biblioteca (pessoal/escolar/comunitária, imagens, chaves de API, dados, GitHub, licença), atualização de biblioteca existente (upload ZIP, extração metadados, migração automática com MIGRATION_MANIFEST)
  • bibfiles.html — Arquivador com 2 abas: verificar assinatura ECDSA P-384 (client-side), assinar e guardar PDF na nuvem (Supabase + Internet Archive + AES-256-GCM), busca por voz, visualização inline, dark mode

2. Como Criar uma Biblioteca

O BibMaker gera automaticamente uma instância personalizada do BibVania, pronta para publicar no GitHub Pages. Não é necessário editar nenhum arquivo manualmente.

⚠️ O arquivo setup_private.sql gerado contém suas chaves de API. Nunca publique esse arquivo no GitHub.
  1. Acesse o BibMaker. Abra o BibMaker no menu de atalhos do site da BibVania (Crie sua Bib) e clique na aba Cadastro de Biblioteca.
  2. Escolha o tipo de biblioteca. Selecione entre Pessoal, Escolar ou Comunitária. O nome do projeto será gerado automaticamente com base nessa classificação.
  3. Preencha os dados do projeto. Nome da biblioteca, autor, site, ano, versão e licença. Esses dados aparecem no rodapé e nas metatags de todas as páginas geradas.
  4. Insira as chaves de API. Supabase URL e Anon Key (obrigatórias), Groq API (para classificação de livros por IA), ImgBB API (para upload de capas) e Internet Archive Access Key + Secret Key (para o BibFiles). As chaves do Groq, ImgBB e Internet Archive são gratuitas. Veja abaixo como criar conta e obter cada chave.
  5. Adicione as imagens (opcional). Logo, favicon e imagem Open Graph. Se não forem fornecidas, o BibMaker usa as imagens padrão.
  6. Informe o repositório GitHub. URL do repositório e da página GitHub Pages onde o sistema será publicado.
  7. Clique em "Criar Biblioteca". O BibMaker gera e baixa um arquivo .zip com todos os arquivos prontos, incluindo README.md, LICENCE.md, VERSION.json, biblog.bak e setup_private.sql.
  8. Configure o Supabase. No painel do Supabase, acesse SQL Editor → New query, cole o conteúdo de supabase_setup.sql e execute. Faça o mesmo com o setup_private.sql. Lembre-se de nunca enviar o setup_private.sql ao GitHub nem compartilhá-lo de nenhuma forma — ele contém suas chaves privadas de API.
  9. Crie o usuário bibliotecário. No Supabase, vá em Authentication → Users → Add user → Create new user, informe e-mail e senha e clique em Create User. Para definir um username (exibido na Área do Bibliotecário e no BibFiles), execute no SQL Editor: INSERT INTO admins (id, nome) VALUES ('uuid-do-usuario', 'Nome do Admin'); — substituindo o UUID pelo valor exibido na lista de usuários do Supabase.
  10. Publique no GitHub Pages. Suba todos os arquivos (exceto setup_private.sql) em um repositório público. Em Settings → Pages → Source, escolha o branch main e pasta / (root).

Como obter cada chave de API

🗄️ Supabase (obrigatório)

  1. Acesse supabase.com e clique em Start your project.
  2. Crie uma conta gratuita com e-mail ou pelo GitHub.
  3. Clique em New Project, dê um nome ao projeto, defina uma senha forte e escolha a região mais próxima (ex: South America — São Paulo). Aguarde a criação — isso pode levar alguns minutos. As tabelas criadas pelo BibVania já têm RLS (Row Level Security) ativado por padrão via supabase_setup.sql; não desative essa opção no painel.
  4. No painel do projeto, vá em Project Settings → API.
  5. Copie a Project URL — esse é o seu Supabase URL.
  6. Copie a chave anon public — essa é a sua Anon Key.

🤖 Groq API (gratuito)

  1. Acesse console.groq.com e crie uma conta gratuita.
  2. Clique em API Keys no menu lateral e depois em Create API Key.
  3. Copie a chave gerada imediatamente — ela não é exibida novamente.

🖼️ ImgBB (gratuito)

  1. Acesse imgbb.com e crie uma conta gratuita.
  2. Vá em About → API ou acesse api.imgbb.com e copie a chave exibida.

📦 Internet Archive (necessário para o BibFiles)

  1. Acesse archive.org, crie uma conta gratuita e confirme o e-mail.
  2. Acesse archive.org/account/s3.php e copie a Access Key e a Secret Key.
  3. No BibMaker, insira as duas no formato access_key:secret_key (separadas por dois-pontos, sem espaços).

🐙 GitHub (obrigatório para publicação)

  1. Acesse github.com e crie uma conta gratuita.
  2. Clique em New repository, dê um nome, marque como Public e clique em Create repository.
  3. Para ativar o GitHub Pages: vá em Settings → Pages → Source, selecione o branch main e a pasta / (root) e clique em Save.

3. Como Atualizar uma Biblioteca

Quando uma nova versão do BibVania é lançada, você pode atualizar sua biblioteca sem perder os dados personalizados. O BibMaker preserva automaticamente suas chaves, imagens e configurações.

💡 O BibMaker detecta automaticamente se a sua versão está desatualizada ao carregar o ZIP antigo.
  1. Acesse o BibMaker. Abra o BibMaker no menu de atalhos do site da sua biblioteca gerada (Crie sua Bib) ou pelo botão Atualizar na área do bibliotecário, e clique na aba Atualização de Biblioteca.
  2. Envie o ZIP da sua biblioteca atual. O BibMaker extrai automaticamente seus metadados e compara com a versão mais recente.
  3. Verifique o relatório de diferenças. O BibMaker exibe quais arquivos serão atualizados e quais dados serão preservados. Se houver campos novos, você será solicitado a preenchê-los.
  4. Clique em "Atualizar Biblioteca". O BibMaker gera um novo ZIP com os arquivos base atualizados e seus dados preservados. São mantidos integralmente: biblog.bak, bibspeech.conf, imagens da biblioteca (logo, favicon, og-image) e todas as configurações do projeto (chaves de API, nome, autor, licença, repositório).
  5. Execute as migrations SQL (se houver). Se a nova versão exigir alterações no banco, o BibMaker indicará os scripts SQL a executar no Supabase.
  6. Publique os arquivos atualizados. Substitua os arquivos antigos no repositório GitHub pelos do novo ZIP (exceto setup_private.sql).

4. Como Usar a Biblioteca

4.1 Leitores

O catálogo público (index.html) não exige cadastro nem login. Qualquer visitante pode pesquisar e consultar o acervo diretamente pelo navegador.

  • Use o menu de atalhos (canto superior esquerdo, ☰) para navegar para a Área do Bibliotecário (se autenticado) ou para abrir o BibMaker em uma nova aba.
  • Clique em ↗ VER ACERVO COMPLETO para abrir a visualização em tabela de todos os livros (abre em nova aba).

Pesquisando livros

  • Use a barra de busca para pesquisar por título, autor ou ISBN.
  • Para buscar por palavras-chave específicas, use # antes do termo (ex: #romance).
  • Clique nas tags coloridas de um livro para filtrar todos os livros com aquela palavra-chave.
  • Use o filtro de categoria para restringir a busca a um tipo de livro específico.
  • Clique em ✕ limpar busca para resetar todos os filtros.

Busca por voz

  • Clique no ícone de microfone na barra de busca para ativar a busca por voz.
  • Fale o termo desejado — o sistema preenche a busca automaticamente.
  • Funciona em navegadores compatíveis com a Web Speech API (Chrome, Edge).

Visualizando um livro

  • Clique na capa de um livro para abrir o lightbox com a imagem em tamanho maior.
  • A disponibilidade do livro é exibida em tempo real no card.
  • Se o livro tiver um arquivo vinculado, um botão de download ficará disponível no card.

Fazendo uma reserva

  • No card do livro desejado, clique em Reservar.
  • Informe seu nome completo — o sistema buscará seu cadastro automaticamente.
  • A reserva fica pendente até ser confirmada ou cancelada pelo bibliotecário na Área do Bibliotecário.
  • Ao confirmar a reserva, o bibliotecário converte-a diretamente em empréstimo — você retira o livro pessoalmente.
  • Reservas não garantem o livro indefinidamente: o bibliotecário pode cancelar se o prazo for ultrapassado ou se outro leitor precisar com urgência.
  • Só é possível reservar livros com pelo menos 1 exemplar disponível no momento da solicitação.
  • Professores e funcionários também podem fazer reservas pelo catálogo público, desde que possuam cadastro no sistema.
  • A reserva só é considerada concluída quando o leitor vai até a biblioteca retirar o livro pessoalmente — é nesse momento que o bibliotecário confirma a reserva e a converte em empréstimo.
  • Alunos só podem ter um livro emprestado por vez: se já houver um empréstimo ativo em seu nome, não é possível fazer uma nova reserva até que esse empréstimo seja concluído (devolvido).
  • Professores e demais funcionários podem ter mais de um livro emprestado simultaneamente, mas só podem fazer uma nova reserva depois de concluir (retirar/confirmar) a reserva anterior — ou seja, não podem acumular reservas pendentes, mesmo podendo acumular empréstimos.

Limitações das reservas

  • O sistema não envia notificações automáticas — o leitor deve acompanhar pessoalmente ou ser informado pelo bibliotecário.
  • Cada leitor pode ter apenas uma reserva pendente por vez — ao fazer uma nova reserva com outra já ativa, o sistema exige confirmar a troca, cancelando a anterior.
  • O botão de reserva é desabilitado automaticamente quando não há exemplares disponíveis — não é possível reservar um livro com todos os exemplares emprestados.
  • Não é possível reservar um livro já emprestado sem nenhum exemplar disponível.
  • Alunos com um livro já emprestado (empréstimo ativo) não podem fazer outra reserva até devolvê-lo — limite de um livro por vez. Essa restrição não vale para professores e demais funcionários, que podem ter vários livros emprestados ao mesmo tempo (mas, como qualquer leitor, só uma reserva pendente por vez).

Acessibilidade

  • Clique no botão 👁️‍🗨️ para ativar o leitor de tela (BibSpeech).
  • Use as setas do teclado para navegar entre os elementos (↑ ← recua, ↓ → avança).
  • Pressione Enter para acionar o elemento em foco.
  • Pressione Espaço para pausar ou retomar a leitura.
  • Pressione L para ler toda a página.
  • Pressione Esc para cancelar a leitura.
  • Pressione Ctrl para abrir ou fechar o leitor a qualquer momento.
  • Pressione ? ou Shift para abrir ou fechar o painel de ajuda com os atalhos disponíveis.
  • O modo escuro pode ser ativado pelo botão 🌙 na barra de acessibilidade.

Acervo completo (acervo.html)

  • Cada livro é exibido como um bloco de texto (título e ID no topo, seguidos de uma linha para cada campo: Autor, ISBN, Categoria, Prateleira, Palavra-chave, Qtd, Cadastro, Capa e Download) — campos sem valor aparecem em branco, sem indicador "—".
  • Clique em [VER CAPA] para abrir o lightbox com a imagem ampliada (feche com Esc ou clique em "Voltar").
  • Clique em [BAIXAR LIVRO] para abrir o arquivo vinculado ao livro, se houver.
  • Exporte os dados em formato de tabela clicando no botão 📥 BAIXAR ACERVO — gera um PDF com 11 colunas (ID, Título, Autor, ISBN, Categoria, Prateleira, Palavra-chave, Capa, Download, Qtd e Cadastro) em layout landscape, paginado e pronto para impressão. A tabela aparece apenas no PDF — a tela usa a lista de texto descrita acima.

Dica de uso

  • Use a busca e os filtros do catálogo para encontrar o livro desejado e verifique sua disponibilidade antes de se deslocar até a biblioteca — o índice mostra em tempo real quantos exemplares estão disponíveis.
  • Use navegadores modernos e atualizados — Chrome e Edge são os mais compatíveis com todas as funcionalidades do BibVania, incluindo BibSpeech e o download de PDFs pelo BibFiles.

4.2 Bibliotecários e Auxiliares de Bibliotecário

O Área do Bibliotecário (admin.html) centraliza todas as operações da biblioteca. O acesso é restrito e exige login com e-mail e senha cadastrados no Supabase.

Entrando no painel

  1. Abra login.html ou clique em Área do Bibliotecário no catálogo público.
  2. Informe e-mail e senha cadastrados no Supabase. Após o login, você será redirecionado ao painel.

Registrando uma devolução

  1. Na aba Devoluções, localize o empréstimo pelo nome do leitor ou pelo título do livro.
  2. Clique no card para expandir os detalhes.
  3. Clique em Registrar Devolução. Um modal pede a data da devolução (preenchida com a data de hoje por padrão, mas editável); confirme para concluir ou cancele para voltar sem alterar nada. A quantidade disponível é atualizada automaticamente.
  4. Para renovar o prazo, clique em Renovar e informe quantos dias adicionais devem ser somados ao prazo atual. Na aba Devoluções, empréstimos sem data definida não podem ser renovados (no card do Acervo o botão fica habilitado e usa a data do empréstimo como base).

Confirmando ou cancelando reservas

  1. Na aba Reservas, as reservas pendentes aparecem com nome do leitor, livro e data de solicitação.
  2. Clique em Confirmar para converter em empréstimo — abre um modal com os dados do livro e do leitor já preenchidos, onde você informa os dias de empréstimo (padrão 7) e a data de início (ou marca "SEM DATA DEFINIDA"); o modal mostra uma prévia da data prevista de devolução antes de confirmar.
  3. Clique em Cancelar Reserva para recusar. A reserva é removida e o livro continua disponível.

Como funcionam as reservas para alunos e professores

Alunos e professores fazem reservas pelo catálogo público (index.html), sem precisar de login. O sistema funciona assim:

  • Alunos: informam o nome completo no momento da reserva. O sistema busca o cadastro automaticamente — por isso é obrigatório estar cadastrado em Pessoas antes de reservar.
  • Professores e funcionários: seguem o mesmo fluxo dos alunos. Devem ter cadastro como funcionário no sistema.
  • A reserva aparece imediatamente na aba Reservas da Área do Bibliotecário com status pendente.
  • Ao confirmar, o bibliotecário registra o empréstimo normalmente — a reserva é removida da fila e o livro é baixado do estoque disponível.
  • Ao cancelar, o livro volta a aparecer disponível para outros leitores.

Limitações do sistema de reservas

  • Não há notificação automática ao leitor — a comunicação é feita pelo bibliotecário (pessoalmente, por mensagem ou outro canal da escola).
  • A primeira reserva tem prioridade: o sistema não permite que dois leitores reservem o mesmo exemplar simultaneamente — quem reservar primeiro fica com a vaga.
  • Uma reserva pendente impede novos empréstimos pelo balcão para o mesmo exemplar esgotado — o bibliotecário deve confirmar ou cancelar a reserva antes de emprestar.
  • Não é possível reservar um livro com 0 exemplares disponíveis: o botão Reservar fica desabilitado nesses casos.
  • Cada leitor pode ter apenas uma reserva pendente por vez — o sistema não permite múltiplas reservas simultâneas.
  • Alunos só podem ter um livro emprestado por vez: se já tiverem um empréstimo ativo, não podem fazer nova reserva até devolvê-lo. Professores e demais funcionários não têm esse limite — podem acumular vários empréstimos ativos, mas, como qualquer leitor, só podem ter uma reserva pendente por vez (precisam concluir a reserva atual antes de fazer outra).

Gerando relatórios

  1. Na aba Relatório, escolha o tipo de período: Anual (intervalo de anos) ou Mensal (intervalo de meses no formato ano-mês). A última escolha é lembrada entre visitas.
  2. O relatório é gerado automaticamente ao abrir a aba ou trocar o período, e é atualizado em tempo real sempre que livros, empréstimos ou pessoas mudam (cadastro, devolução, renovação, reserva confirmada/cancelada etc.).
  3. O relatório traz: resumo geral (títulos, exemplares, empréstimos no período, reservas, pessoas cadastradas); livros mais lidos pelos alunos; livros mais renovados; alunos que mais leram (geral, meninos e meninas); e funcionários que mais leram. As seções de alunos têm subtópicos por segmento — Ensino Fundamental I, Fundamental II, Ensino Médio e EJA.
  4. Cada seção e subtópico pode ser expandido/recolhido clicando em seu título; abrir uma seção recolhe as demais no mesmo nível, mantendo o relatório organizado.
  • O painel atualiza as informações em tempo real via Supabase Realtime — não é necessário recarregar a página.
  • Ao cadastrar um livro com IA, revise os campos antes de salvar — a IA pode ocasionalmente errar a categoria.
  • Para cadastrar múltiplos exemplares, aumente o campo Quantidade total ao invés de cadastrar o livro mais de uma vez.
  • O campo de prateleira aceita número ou barra (ex: 3 ou 3/8).
  • Registre devoluções imediatamente ao receber o livro físico — o sistema atualiza a disponibilidade em tempo real para todos os leitores no catálogo público.
  • Para excluir um livro, ele precisa estar sem empréstimos ativos.

4.3 Desenvolvedores

Esta seção é voltada para quem deseja entender, modificar ou manter o código-fonte do BibVania. O projeto é 100% frontend (HTML + CSS + JavaScript vanilla) com backend via Supabase (PostgreSQL + Realtime + Auth + Storage).

Criando um projeto no Claude

A forma recomendada de trabalhar no BibVania com assistência de IA é criar um Projeto no Claude (claude.ai/projects). Isso permite que o contexto do projeto seja preservado entre conversas.

  1. Acesse claude.ai/projects e crie um novo projeto chamado BibVania.
  2. Faça upload do arquivo bibinit.claude nas instruções do projeto — ele contém as regras arquiteturais imutáveis que a IA deve seguir.
  3. Em cada conversa, faça upload do ZIP do projeto atualizado. O Claude lerá todos os arquivos, verificará a assinatura bibhash.sig e exibirá o painel de recepção.
  4. Ao receber o ZIP modificado de volta, substitua os arquivos alterados no seu repositório GitHub.

Importante: o arquivo bibinit.claude é o coração do projeto — ele define regras que não podem ser violadas nem mesmo por instrução do usuário. Nunca o modifique sem entender o impacto em toda a arquitetura.

bibhash.sigAssinatura SHA-256 de integridade do ZIP do projeto BibVania. Formato: SHA-256 hex puro (64 caracteres), sem prefixo, sem newline. Distinto da assinatura de biblioteca gerada pelo BibMaker (embutida no ZIP de cada biblioteca, cobre apenas aquele ZIP específico); ambas usam o mesmo formato SHA-256 hex puro de 64 caracteres.

Arquitetura técnica resumida

  • Dois sistemas de script por página: cada HTML carrega bibcore.js como ES Module (<script type="module">) e um bloco UMD síncrono logo após. Funções acessadas via onclick devem ser expostas em window.* pelo módulo.
  • Transição de página: função window.navegarCom(url) exibe um overlay fixo com logo (80×auto, animação de escala 0.6s), 3 bolinhas (10×10px, animação de salto 0.9s com delays escalonados de 0.3s), fundo com cor var(--bg), opacity 0, pointer-events none. Ao chamar navegarCom(), a classe ativo liga opacity/pointer-events, aguarda 140ms e redireciona. No evento pageshow (inclui bfcache do navegador), remove a classe para fade-out 0.1s.
  • BibMaker redirect: window.abrirBibMaker(dest?, aba?) abre nova aba com a tela de transição visível (callback mostrar()) e define timeout de 3s antes de redirecionar. dest padrão é 'bibmaker.html' (relativo); em bibliotecas geradas, o BibMaker replace via modificarBibvaniaJs() com URL absoluta do repositório. Se aba for passado (ex: 'cadastro', 'atualizacao'), adiciona ?aba=<aba> à URL.
  • Inicialização sem flash: o <body> começa com visibility:hidden. Um script inline no <head> restaura o tema e lê VERSION.json de forma síncrona antes do CSS carregar. O body só fica visível após tudo estar pronto.
  • RLS no Supabase: leitura pública (anon) para livros, pessoas, emprestimos e reservas. reservas também permite inserção pública (anon WITH CHECK (true)) — alunos reservam sem login. Escrita completa exige autenticação (auth.role() = 'authenticated'). config_privada só permite leitura autenticada (sem INSERT/UPDATE/DELETE via API). admins é restrita ao próprio usuário (auth.uid() = id). bibfiles_chaves é restrita ao próprio usuário. bibfiles_pdfs permite leitura pública (anon) para verificação de assinaturas, mas escrita exige autenticação. livros_excluidos exige autenticação para leitura e escrita.
  • Trigger automático: a função recalcular_disponivel() no PostgreSQL recalcula quantidade_disponivel após qualquer mudança em emprestimos. Nunca atualizar esse campo manualmente.
  • Paginação automática: DB._fetchAll() contorna o limite nativo de 1000 registros por requisição do Supabase, paginando automaticamente em lotes para exibir o acervo completo.
  • Realtime: 4 canais ativos (livros-changes, emprestimos-changes, pessoas-changes, reservas-changes) monitoram eventos INSERT, UPDATE e DELETE em suas tabelas respectivas. Fallback de polling a cada 60 segundos se a conexão WebSocket falhar. Cada canal é subscrito via DB.on<Tabela>Change(callback).

Convenções e padrões internos

Estas convenções são definidas em bibinit.claude e devem ser preservadas em qualquer modificação no código:

  • Campos case-sensitive: imagem_url, pdf_url, alt_text e foto_url nunca são convertidos para maiúsculas. Todos os demais campos (título, autor, categoria, prateleira etc.) são padronizados com toUpperCase() pela função DB.padronizarObjeto(obj) em bibcore.js antes de qualquer INSERT/UPDATE. A função opera sobre objetos simples (livros, pessoas, empréstimos) preservando URLs e normalizando apenas os campos de texto.
  • Palavras-chave normalizadas: o array palavras_chave é normalizado em minúsculas, sem espaços nas pontas e limitado a 25 caracteres por termo (.toLowerCase().trim().slice(0,25)). A busca por tag usa a sintaxe #palavra no catálogo público.
  • Autenticação (checkAuth + refreshSession): antes de qualquer operação em admin.html e bibfiles.html, o sistema chama DB.checkAuth(). Se a sessão estiver nula, tenta DB.refreshSession() para renovar o token via refresh_token do localStorage; se falhar, redireciona para login.html. Nunca usar getSession() isolado — um JWT expirado faz requisições como usuário anônimo e pode causar erros silenciosos.
  • Sistema de abas: a aba ativa de cada painel é salva em sessionStorage (chaves: bibvania_tab, bibmaker_tab, bibfiles_tab). Um script inline no <head> restaura a aba correta antes do DOMContentLoaded, evitando o "flash" da aba padrão. A URL também aceita o parâmetro ?aba=<nome> para abrir diretamente em uma aba específica.
  • Chaves de localStorage padronizadas: theme (claro/escuro), bibvania_admin_nome, bibvania_tab, bibmaker_tab, bibfiles_tab, bibvania_signout_intencional e supabaseSession (JSON com access_token e refresh_token). Novas chaves não devem ser criadas sem necessidade — isso pode quebrar a persistência entre sessões.
  • Navegação com transição: links e botões que mudam de página usam window.navegarCom(url) em vez de window.location. A função exibe uma sobreposição com a logo animada durante a troca de página, dando uma sensação de SPA. Não deve ser usada para navegações internas (abas) — apenas para mudanças de página.
  • Carregamento de imagens com nova tentativa: logo, favicon e demais imagens fixas (og-image, ícones do BibMaker) tentam carregar até 3 vezes, com atraso crescente (100ms, 500ms, 1000ms). Se todas falharem, exibe um texto alternativo em vez de deixar a página com um ícone quebrado. Implementado via loadImageWithRetry(url, maxRetries, delays).
  • Upload de capas e PDFs: DB.uploadCapaLivro(livroId, file) envia a imagem para ImgBB (obtém chave via config_privada.imgbb_api_key) e retorna a URL; DB.uploadPdfLivro(livroId, file) usa o Internet Archive para armazenamento permanente (chave via config_privada.internetarchive_api_key). Remoção: DB.removerCapa(livroId) e DB.removerPdf(livroId) deletam as URLs das colunas imagem_url e pdf_url respectivamente (não deletam o arquivo na origem).
  • Badges de categorias: função spanCategoria(nome, cor?) retorna um <span> com estilos inline: background var(--bg, #0f172a), cor do texto da tabela de categoria_cores, border-radius 4px, padding 4px 8px. Se cor não for passado, busca na tabela categoria_cores; se não encontrar, usa #666.
  • Acessibilidade (WCAG 2.1): listas que mudam dinamicamente (ex: livros adicionados/removidos no catálogo) usam aria-live="polite" para anunciar a mudança a leitores de tela. Modais e o lightbox de capas implementam focus trap (o foco do teclado não escapa do modal) e podem ser fechados com Esc. Busca por voz via startVoiceSearch() (Web Speech API, idioma pt-BR).
  • Sistema de migração do BibMaker (MIGRATION_MANIFEST): objeto em bibmaker.html que lista, para cada par de versões from→to do BibVania, os campos de configuração novos exigidos do usuário ao atualizar uma biblioteca antiga (estrutura por campo: id, label, helpText, placeholder, type, required, extractFrom: {file, regex} para extração automática do ZIP antigo, injectInto: [{file, marker}] para substituir marcadores %%MARKER%% nos arquivos base). Toda vez que uma nova versão do BibVania introduzir um campo de configuração novo (chave de API, URL, flag etc.), deve-se adicionar uma entrada em migrations — entradas antigas nunca são removidas, pois usuários podem pular versões. A versão exibida do BibMaker (BIBMAKER_VERSAO) é derivada automaticamente da versão do BibVania como major.(minor-3) (ex.: BibVania v1.8 → BibMaker v1.5).

O BibSpeech pode ser instalado em qualquer site externo sem depender do restante do BibVania. Basta copiar três arquivos e adicionar um botão:

  1. Copie bibspeech_api.js, bibspeech.conf e bibspeech-logo.png para a raiz do projeto de destino.
  2. Inclua o script antes do </body>: 
    <script src="bibspeech_api.js"></script>
  3. Adicione um botão para abrir o leitor: 
    <button id="leitorTelaBtn" onclick="abrirLeitorTela()">Leitor de tela</button>

O script injeta automaticamente o CSS, o painel e o popup no DOM — nenhuma configuração adicional é necessária. O idioma e as pronúncias são controlados pelo bibspeech.conf.

Backup do Supabase

O BibVania armazena todos os dados da biblioteca (livros, pessoas, empréstimos, reservas e configurações) no Supabase. Faça backup regularmente para evitar perda de dados em caso de exclusão acidental, problema na conta ou encerramento do projeto gratuito.

  1. Acesse o painel do seu projeto em supabase.com/dashboard.
  2. No menu lateral, vá em Table Editor e exporte cada tabela importante (livros, pessoas, emprestimos, reservas) clicando em Export CSV.
  3. Alternativamente, use o SQL Editor para exportar via COPY ou acesse Database → Backups (disponível nos planos pagos) para fazer download de um dump completo.
  4. Guarde os arquivos exportados em local seguro (nuvem pessoal, HD externo) e repita o processo ao menos uma vez por semana, ou sempre antes de grandes alterações no sistema.

Importante: o plano gratuito do Supabase não inclui backups automáticos com restore por interface. Projetos inativos por mais de 7 dias podem ser pausados automaticamente — acesse o painel e reative o projeto antes que os dados sejam excluídos.

Segurança das chaves de API

O BibVania utiliza chaves de API para integrar serviços externos (Groq, ImgBB, Internet Archive). Siga estas boas práticas:

  • Nunca adicione chaves de API diretamente em arquivos HTML ou JS que vão para o repositório público. Use a tabela config_privada no Supabase para armazená-las com segurança (conforme o arquivo setup_private.sql).
  • Rotacione as chaves periodicamente ou sempre que suspeitar de uso indevido.
  • No painel de cada serviço, configure limites de uso (rate limits) para evitar cobranças inesperadas.

Cuidados com a conta GitHub

  • Use uma senha forte e ative a autenticação em dois fatores (2FA) na sua conta GitHub.
  • Mantenha um e-mail de recuperação atualizado.
  • Evite tornar o repositório privado sem antes verificar se o GitHub Pages ainda funcionará (Pages gratuito exige repositório público ou plano pago).
  • Faça um clone local do repositório periodicamente para ter uma cópia de segurança dos arquivos HTML, JS e CSS.
  • Verifique periodicamente em Settings → Pages se o site continua ativo — o status deve aparecer como Your site is live.

Monitoramento

  • Verifique mensalmente o uso do banco de dados no painel do Supabase. O plano gratuito tem limite de 500 MB de armazenamento e 50.000 requisições por mês.
  • Ative notificações por e-mail no Supabase para ser avisado sobre pausas automáticas do projeto.
  • Antes de aplicar qualquer alteração em produção, teste em uma instância separada do BibVania (criada com o BibMaker usando outro projeto Supabase). Nunca execute scripts SQL em produção sem antes validar em ambiente de teste.
